A maioria das contas é provisionada automaticamente quando o RH cria seu cadastro de pessoa colaboradora. Esta página lista o que você deve ter e como conseguir acesso se algo estiver faltando.

## Contas provisionadas automaticamente

Estas são criadas antes da sua data de início e ativadas quando você faz login com seu e-mail corporativo:

| Ferramenta | Usada para |
|------------|-----------|
| E-mail corporativo | Comunicação principal |
| Slack | Mensagens entre times |
| Rastreador de projetos | Gestão de tarefas e sprints |
| Portal do RH | Benefícios, folgas, folha de pagamento |
| SSO-easy | Single sign-on para os outros apps |
| PasswordBuddy | Gerenciador de senhas |

## Acesso sob solicitação

Alguns sistemas exigem solicitação separada, seja por serem específicos do cargo ou por terem limite de licenças:

| Ferramenta | Como solicitar |
|------------|---------------|
| Infraestrutura | Abra uma [solicitação de acesso ao TI](/pt-br/it-security/access-requests) |
| Repositório Git | Peça ao seu tech lead para te convidar |
| Plataforma de design | Peça à pessoa lead de design |
| Ferramentas de analytics | Solicite via ticket de TI, com aprovação do gestor |
| Sistemas com dados de cliente | Aprovação do gestor + segurança obrigatória |

## Algo faltando?

Abra um [ticket de solicitação de acesso](/pt-br/it-security/access-requests) e inclua:
- A ferramenta que precisa
- O motivo
- O nome do seu gestor para aprovação

Solicitações rotineiras são atendidas em até um dia útil.
